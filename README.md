# Object_Oriented_Paradigm-Car_Comparison_Game
A design of a "Top Trumps" style card game, using car data inspired by carsales.com.au for the statistics.

## Process:
1. Create Cards (Attributes, Design) - Reason: Designing the cards is important to do correctly as the game revolves around them working as intended.
2. UML Class Diagram (Classes, Structure) - Reason: This contains the remaining parts of the game. The classes and relationships between them are essential for the game to function.
3. Game Mechanics (Rules) - Reason: The rules of the game are essential to get correct so that the game can function as intended.
4. Structure Chart - Reason: Creating a structure chart provides a clear representation of how the program functions and how each interaction occurs.
5. Social, Ethical and Legal Implications - Reason: This step is the final one as it involves the least relation to the creation of the game. This is meant to be reserved for reflection on how the game could be improved.
